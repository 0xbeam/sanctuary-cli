parc

Agent-first architecture intelligence

$ npm i -g parc click to copy

Then run parc map ./your-repo

How it works
01
Drop a repo
Point parc at any git repo. No config, no setup, no dependencies.
02
Instant analysis
Routes, APIs, agents, components, database schema — detected in seconds.
03
Interactive diagrams
One HTML file. Searchable, filterable, shareable. Always current.
Commands
parc map <repo>
Architecture Map
Generate an interactive HTML diagram. Routes, APIs, agents, components, database tables, and dependencies — all in one view.
parc serve <repo>
Live Dashboard
Real-time dashboard with file watching and auto-rescan. Stats, agent visualization, and a rescan button for standups.
parc agents <repo>
Agent Registry
Every AI agent in your codebase. Model tier, tools used, descriptions. Know exactly what your agents do.
parc diff [ref]
Architecture Diff
How your architecture changed since any commit. Added routes, modified agents, new endpoints — grouped by concern.
What it detects
Frameworks
Next.js, Vite, SvelteKit, Express
Framework, version, and features like Tailwind, Supabase, Prisma, Zod.
Routes & APIs
App Router, Pages, REST
Every route and API endpoint with HTTP methods and auth detection.
AI Agents
Model tiers & tools
Reasoning vs extraction tiers, tools used, and descriptions per agent.
Components
UI & feature split
Base UI components separated from feature-level compositions.
Database
Tables & migrations
Prisma schemas and SQL migrations mapped to your data model.
Git
Branch, commits, remote
Current branch, last commit, author, commit count, and remote URL.
Why parc
Manual documentation
  • Diagrams go stale within a week
  • No single source of truth for agents
  • Grep through code to find endpoints
  • Onboarding takes days of exploration
Hours per update
With parc
  • Always-current diagrams from source
  • Every agent cataloged with tier and tools
  • All routes and endpoints mapped
  • New team members productive in minutes
